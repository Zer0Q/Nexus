# Connection-Capture-Convention

## Definition
A capture convention that requires noting the connection that made something worth capturing and the first use case that comes to mind at the moment of capture. Format: [THE IDEA] CONNECTS TO: [what this relates to] MIGHT USE FOR: [first use case].

## Why It Matters
Preserves the context of relevance present at capture time, which fades rapidly. Takes thirty additional seconds but doubles the value of the note because the processing workflow has immediate hooks to work with.

## Key Ideas
- Note the connection that triggered the capture
- Note the first use case that comes to mind
- These two fields give the processing workflow immediate context for assessment and connection
- Without this, the system must infer relevance from content alone, which is less reliable

## Tradeoffs
- Adds friction to capture; may discourage rapid note-taking in flow states
- The "might use for" field may become stale if the use case changes
