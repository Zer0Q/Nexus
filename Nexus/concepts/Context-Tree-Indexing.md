# Context Tree Indexing

## Definition
A hierarchical indexing method where knowledge is organized as a tree of contexts rather than flat tags. Each node in the tree represents a context (topic, project, domain), and notes are linked to the most specific context that applies.

## Why It Matters
Provides structured navigation without the rigidity of folder hierarchies. Enables both broad overviews and deep dives into specific contexts, bridging the gap between flat wikilinks and rigid folder trees.

## Key Ideas
- Tree structure mirrors how humans categorize information naturally
- Each context node can serve as an index page for that domain
- Used by tools like ByteRover to navigate knowledge bases programmatically
