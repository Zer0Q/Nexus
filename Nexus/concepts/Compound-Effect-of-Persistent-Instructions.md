# Compound Effect of Persistent Instructions

## Definition
AI behavior instructions stored in persistent files accumulate value over time — each rule added prevents future mistakes, and the total benefit grows non-linearly as the instruction set matures.

## Why It Matters
Most AI interactions are stateless — every session starts from zero. Persistent instructions flip this: week 1 fixes generic output, week 2 adds pattern-specific rules, week 3 specializes per project, week 4 enables hands-off delegation. The investment is linear (adding rules), but the payoff compounds (fewer corrections, faster iteration, consistent quality).

## Key Ideas
- **Week 1:** Claude stops giving generic output; every response matches style and constraints
- **Week 2:** Instruction set grows from 10 to 40 lines as patterns emerge from real usage
- **Week 3:** Different configs per project — content, research, code — each creates a specialist
- **Week 4:** Shift from chatbot to task delegation; output matches standards because standards are baked in
- The compound effect is permanent — unlike prompt engineering which resets each session

## Tradeoffs
- Maintenance cost: instructions need periodic review as projects evolve
- Over-constraint risk: too many rules can make the AI rigid and uncreative
- Portability: project-specific instructions don't transfer between contexts
