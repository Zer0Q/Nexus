# Semantic Model

## Definition
Formal structures for knowledge representation -- knowledge graphs, ontologies, and schemas -- that provide a deterministic substrate for AI reasoning, contrasting with the probabilistic nature of LLMs.

## Why It Matters
LLMs excel at creative association and NLP but hallucinate. Semantic models impose logical constraints, validate structural relationships in real time, and provide persistent, structured, fully auditable memory. They anchor probabilistic model outputs in verifiable data structures.

## Key Ideas
- **Shared vocabulary and persistent identifiers** -- Base concepts (Client, Order, Resource, State) that all systems reference consistently
- **Symbolic AI substrate** -- Formal framework for deductive and inductive processing, complementing connectionist AI (neural networks)
- **Dynamic evolution** -- No longer static lookup databases; agents continuously use and generate semantic models as working memory
- **Enterprise unifier** -- A shared semantic layer connects each system once to shared business concepts, transforming integration complexity from N(N-1) exponential to N linear scaling
- **Three roles in agentic AI** -- Structured brain for agents, working memory during execution, and the autonomous product agents generate

## Tradeoffs
- Manual ontology design requires intensive expert effort (mitigated by Zero-Shot Ontology approaches)
- Over-constraining can limit creative inference capabilities of LLMs
- Maintenance burden grows with domain complexity
