# Enterprise Context Layer

## Definition
The system that turns knowledge, expertise, and norms into machine-usable context for AI agents across an enterprise. It serves as a shared enterprise brain -- the foundation every agent reaches into, learns from, and contributes back to, enabling the tenth agent to be dramatically smarter than the first.

## Why It Matters
Without a shared context layer, each AI agent starts from scratch, re-deriving business knowledge independently. The context layer enables compounding learning -- every interaction improves the substrate, and every new agent inherits accumulated knowledge, expertise, and governance.

## Key Ideas
- Three kinds of context: Knowledge (business map), Expertise (how work gets done), Norms (rules of acceptable action)
- Core substrate: AI-ready data + knowledge graph, semantics/ontology, skills
- Five capabilities: context mining, development lifecycle, compounding learning loops, activation/retrieval, governance
- Memory split: working/episodic near agent harness; semantic/procedural in context layer
- Context islands (separate layers per team) break the compounding loop
- Governance concerns: quality, drift, lineage, versioning, approval

## Tradeoffs
- Centralized context layer vs distributed team-owned context
- Breadth (covering all domains) vs depth (domain-specific richness)
- Automated mining vs human-curated context
