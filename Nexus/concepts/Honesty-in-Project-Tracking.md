# Honesty in Project Tracking

## Definition
The principle that project status should reflect actual recent activity rather than database fields that create the illusion of organisation. A project note either has recent activity or it does not -- no "In Progress" labels that lie.

## Why It Matters
Database status fields (Notion, Trello, etc.) encourage false organisation. You mark everything "In Progress" and never update it. Plain text project notes are honest: if you have not written in them recently, they are stale.

## Key Ideas
- Status fields create illusion of organisation without substance
- Plain text notes are self-documenting through recency
- No database views needed -- recent activity IS the status
- Reduces maintenance overhead of keeping status fields updated
