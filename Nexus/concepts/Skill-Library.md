# Skill Library

## Definition
Colección organizada de skills que representa el manual operativo de una empresa en formato ejecutable por agents. No es solo un repositorio técnico — es el activo estratégico que diferencia a una empresa: captura juicio específico, procesos propietarios y estándares de calidad acumulados con el tiempo.

## Why It Matters
Dos empresas usando el mismo modelo frontier tienen resultados diferentes cuando una tiene una skill library y la otra solo connectors. La skill library convierte conocimiento tácito en ventaja operativa repetible. Cada skill es una pieza de palanca operacional que eleva el piso para todos y captura juicio que antes tomaba años construir.

## Key Ideas
- **Skill > prompt:** un prompt es puntual, un skill es reutilizable — captura pasos, ejemplos, edge cases y criterios de calidad
- **Privado > genérico:** los skills más valiosos son específicos de empresa (proceso de escalación, criterios de calificación, formato de reportes). No se descargan de marketplaces
- **Evolución del stack:** connectors (acceso) → skills (juicio) → plugins (acceso + acción). Los skills son la capa que falta entre datos y acción
- **Materia prima:** el conocimiento de los mejores performers — qué miran primero, qué pasan por alto los demás, qué errores evitan
- **Patrón histórico:** Unix commands → scripts → libraries → APIs → workflows → skills. Cada capa hizo reutilizable una forma diferente de conocimiento
- **El executor cambió:** antes los humanos leían el playbook, ahora los agents lo cargan, inspeccionan archivos, ejecutan scripts y continúan. El playbook se vuelve activo

## Tradeoffs
- Skills genéricos de marketplaces son punto de partida pero no ventaja competitiva
- Mantener skills actualizados requiere owners cercanos al trabajo — skills huérfanos se degradan
- El esfuerzo inicial de mapear trabajo repetitivo y documentar juicio es no trivial
