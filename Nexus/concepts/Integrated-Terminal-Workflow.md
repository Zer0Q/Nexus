# Integrated Terminal Workflow

## Definition
Running the AI agent's terminal as a tab inside Obsidian, eliminating window switching between notes and agent. The agent generates .md files that appear in Obsidian automatically.

## Why It Matters
Interface friction kills adoption. When the terminal fights you (separate window, alt-tab, lost context), you stop using the system. Integrated terminal makes the agent feel like a native part of the vault.

## Key Ideas
- Obsidian Terminal plugin in integrated mode
- Agent as a tab inside your notes -- same window, same context
- "You stop opening apps. You stop creating files manually. You just talk to the terminal."
- Agent generates .md file in right place, opens it in Obsidian automatically
- Obsidian's command line access gives agent hands -- direct access to Obsidian functions

## Tradeoffs
- Terminal plugin may have limitations vs. full terminal
- Debugging agent issues harder in integrated mode
- May not support all terminal features (colors, interactive tools)
