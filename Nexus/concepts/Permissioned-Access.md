# Permissioned Access

## Definition
Gated access to frontier AI where users can consume intelligence but cannot freely bootstrap independent systems from it. The asymmetry: the provider learns from everything (internet, books, code, user data), but users cannot freely learn from the provider's outputs to build competing systems.

## Why It Matters
When the permission gate sits between builders and the capability to reason, automate, code, and design, access becomes a class hierarchy. Large labs get the best models, strategic partners get trusted access, startups get API access until they compete, everyone else gets refusals and a monthly bill.

## Key Ideas
- Anthropic's ToS: customers own outputs but cannot use them to train or develop competing AI models without written permission
- Prohibited examples include general-purpose chatbots, open-ended text generation, AI assistants, coding assistants, translation systems
- The plantation model for cognition: rent the tool, generate value, do not build the successor
- Five-year data retention for training (user workflows improve Anthropic) while users cannot freely train competitors on Claude outputs — learning flows up, independence does not flow back
- Claude's Constitution makes Anthropic's values the root behavioral authority — Claude is Anthropic's agent rented to you

## Tradeoffs
- Providers argue output-use restrictions prevent safety controls from being lost in competing models
- The boundary of "competing systems" is vague by design — as the provider absorbs more user data, more activities can be reframed as competitive
- Opensource AI depends on the freedom to build, reproduce, modify, distill, benchmark, and create competing systems
