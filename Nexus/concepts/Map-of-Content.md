# Map of Content (MOC)

## Definition
A note whose primary purpose is to link to other notes rather than contain original ideas. An index hub for a cluster of 20+ related notes, making an entire topic navigable from a single starting point.

## Why It Matters
As vaults grow from hundreds to thousands of notes, flat search becomes insufficient for orientation. MOCs provide topic-level navigation without moving notes into folders -- they link, not contain.

## Key Ideas
- Create when a topic has accumulated 20+ notes
- Sections: core framework notes, tool notes, book notes, project applications, open questions
- MOC is not a folder -- you link to notes, not move them
- Multiple notes can appear in multiple MOCs (many-to-many relationships)
- Obsidian's graph view shows MOCs as hub nodes
- AI can generate MOCs from existing note clusters

## Tradeoffs
- MOCs themselves need maintenance as notes are added/removed
- Too many MOCs creates navigation paralysis
- Risk of MOC becoming stale if not updated
