# PARA Method

## Definition
Organizational framework with four categories: Projects (active work with defined outcomes and end dates), Areas (ongoing responsibilities without end dates), Resources (reference material by topic), Archive (everything no longer active).

## Why It Matters
Provides a clear distinction between active work and reference material. Projects have deadlines; areas are ongoing; resources are topical references; archive captures everything without deleting. Prevents the "where does this go?" problem.

## Key Ideas
- Projects: defined outcome, end date, moves to archive when complete
- Areas: ongoing responsibilities (health, finances, career) -- never complete
- Resources: personal Wikipedia -- topics, people, places, tools
- Archive: everything inactive -- storage is cheap, accidental deletion is not
- Combined with INBOX (processing queue) and SYSTEM (vault infrastructure) = 7 folders
- 5-8 top-level folders max

## Tradeoffs
- Notes spanning multiple areas create ambiguity
- Requires regular archiving to keep active folders manageable
- Not all content fits neatly into one category
