# Goal-Driven Agents

## Definition
Agents that operate based on declarative goals rather than imperative commands — the user states what they want, and the agent figures out how to achieve it through planning and execution.

## Why It Matters
Imperative prompting requires the user to know the steps. Goal-driven agents shift the burden from "how" to "what," enabling non-technical users to leverage complex workflows.

## Key Ideas
- /goal command sets declarative objectives
- Agent breaks goals into subtasks
- Agent executes subtasks autonomously
- Progress tracking and mid-goal steering
- Goal persistence across sessions

## Tradeoffs
- Agent may choose suboptimal paths
- Hard to predict exactly what the agent will do
- Goal ambiguity leads to wasted effort
