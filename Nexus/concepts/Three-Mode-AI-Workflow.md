# Three-Mode AI Workflow

## Definition
A workflow pattern that categorises AI tasks into three modes based on data sensitivity: (1) fully offline for deep work, (2) hybrid for research and writing, (3) strict local for confidential client work.

## Why It Matters
Not all tasks have the same privacy requirements. A rigid local-only or cloud-only approach is suboptimal. Three-mode workflow lets you match the tool to the task.

## Key Ideas
- Mode 1 (Deep work): LM Studio + local model, fully offline
- Mode 2 (Research): Obsidian local + cloud AI for heavy reasoning
- Mode 3 (Confidential): Strictly local, vault stays private
- Mode selection is per-task, not per-session
- Requires clear mental boundaries about data sensitivity
