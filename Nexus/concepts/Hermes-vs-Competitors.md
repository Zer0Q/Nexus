# Hermes vs Competitors

## Definition
Positional comparison of Hermes Agent against OpenClaw, Claude Code, and Codex. Each tool serves a different job — the key insight is using them complementarily rather than as replacements.

## Why It Matters
Understanding where each tool fits prevents tool fatigue and ensures you reach for the right instrument. Hermes is the general-purpose employee; Claude Code/Codex are for deep focused coding sessions.

## Key Ideas
- **Hermes vs OpenClaw**: Hermes is lighter, snappier, updates don't break setup. Built-in multi-agent via Kanban, Nous Portal with curated models, 166 tracked skills across 26+ categories, 20+ messaging platforms.
- **Hermes vs Claude Code/Codex**: Different jobs, use both. Hermes = general purpose employee (research, documents, admin, business advisor, prototypes). Claude Code/Codex = deep focused vibe coding, large-scale apps, end-to-end testing.
- Hermes handles everything else so you can go deep when needed
- "Chief of Staff" metaphor: Hermes manages the day-to-day, you focus on what requires deep work

## Tradeoffs
- Hermes may not match Claude Code's coding depth for very large applications
- Running multiple tools means managing multiple configurations
- Context switching between tools has a cognitive cost
