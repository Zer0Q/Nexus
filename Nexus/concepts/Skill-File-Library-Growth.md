# Skill File Library Growth

## Definition
The compounding growth of agent-authored skill files over time. In six weeks of connected Hermes-Obsidian operation, the skill library grew from zero to 23 entries, with every recurring workflow executed more than twice written into a reusable skill.

## Why It Matters
Skill growth is the measurable output of agent self-improvement. Each skill represents a workflow the agent has solved, documented, and optimized. The agent executes these workflows faster and more accurately each time because it loads proven procedures instead of re-figuring approaches from scratch.

## Key Ideas
- Week 1-2: setup and calibration, few skills created
- Week 4: morning brief produces genuinely surprising output
- Week 6: 23 skills, every recurring workflow automated
- Skills are SKILL.md files with YAML frontmatter (name, description, version, author)
- The compounding effect: Hermes writes to the vault as well as reads, accelerating growth
- Growth rate depends on task complexity and user engagement

## Tradeoffs
- Agent may save suboptimal approaches (self-congratulation bias)
- Skills can overwrite manual customizations
- Requires maintenance to prevent skill bloat
