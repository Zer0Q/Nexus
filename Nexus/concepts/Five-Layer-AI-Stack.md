# Five-Layer AI Stack

## Definition
A five-layer AI architecture where each operational layer is owned by a different tool: reasoning (Claude), memory (Obsidian), automation (Hermes Agent), coding (Kimi K2.6), and execution (Cursor). Each layer feeds the next in a coordinated pipeline.

## Why It Matters
No single tool is the answer to everything. The operators getting real leverage run all five in coordination. Each tool does something the others cannot replicate, and the combination produces capabilities no single tool can achieve alone.

## Key Ideas
- **Layer 1 -- Reasoning**: Claude. Everything flows through it for thinking, writing, and analysis. 200K context window, superior instruction following.
- **Layer 2 -- Memory**: Obsidian. Holds accumulated context that makes Claude's outputs compound over time rather than starting fresh every session.
- **Layer 3 -- Automation**: Hermes Agent. Runs recurring workflows -- daily briefs, scheduled research sweeps, automated reports.
- **Layer 4 -- Coding**: Kimi K2.6. Handles large-scale coding tasks and multi-agent orchestration requiring parallel execution at scale.
- **Layer 5 -- Execution**: Cursor. Live coding work inside your actual codebase with visual context and parallel agents in real git branches.

## Tradeoffs
- Five tools means five integrations, five configuration surfaces, five potential failure points
- Each tool has its own learning curve
- Coordinating the stack requires understanding each tool's strengths and limitations
