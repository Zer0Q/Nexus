# Feedback Loop Knowledge System

## Definition
A knowledge system where information that goes in must come back out -- the vault pushes insights to the user rather than waiting for the user to pull them. Solves the graveyard problem: a second brain that never talks back is just an organized way to forget things.

## Why It Matters
Most knowledge systems fail because they're designed for input, not output. Capture everything, retrieve nothing. The vault grows, thinking does not. A feedback loop ensures the vault actively surfaces connections, patterns, and questions.

## Key Ideas
- Three failure modes: capture friction, no connection layer, no reason to return
- Daily brief pushes insights before you open anything else
- Weekly synthesis builds thesis from accumulated captures
- Contradictions section forces thinking updates
- "Information that goes in but never comes back out is not a knowledge system. It is a graveyard with good folders."

## Tradeoffs
- Push notifications can become noise if not calibrated
- Requires trust in AI filtering of what's "important"
- Balance between push (daily brief) and pull (on-demand queries)
