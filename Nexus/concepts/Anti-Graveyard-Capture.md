# Anti-Graveyard Capture

## Definition
A capture system designed so that saved information does not die in an unorganised list or folder. The key mechanism: every captured item enters a processing pipeline where it will be reviewed, connected, or discarded within a known timeframe.

## Why It Matters
Bookmark graveyards are the default outcome of most saving systems. X bookmarks, browser folders, and reading lists accumulate hundreds of items that are never revisited. Anti-graveyard design ensures every capture has a fate.

## Key Ideas
- Bookmark folders look like organisation but function like cemeteries
- Every capture must enter a processing pipeline, not a storage folder
- Processing can be manual (weekly review) or automated (AI synthesis)
- Items that do not get processed should be discarded, not hoarded
- Fast capture + scheduled processing = anti-graveyard

## Tradeoffs
- Requires a processing mechanism (otherwise inbox becomes the new graveyard)
- May feel wasteful to discard items that were once interesting
