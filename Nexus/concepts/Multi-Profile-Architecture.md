# Multi Profile Architecture

## Definition
Running multiple isolated AI agents in parallel, each with its own identity (SOUL.md), memory, skill library, and configured model. Profiles specialize independently as they accumulate skills, converging on an org-chart pattern where each profile mirrors a team role.

## Why It Matters
A single agent forced to handle coding, research, content, and finance accumulates conflicting conventions and diluted expertise. Separate profiles let each agent master its domain, build specialized skills, and maintain clean memory boundaries.

## Key Ideas
- Each profile has isolated SOUL.md, memory, skills, and model configuration
- Converging pattern: org chart profiles (Chief of Staff, Head of Research, Head of Content, Head of Finance)
- Chief of Staff: knows high-level priorities, runs daily brief
- Head of Research: drills deep on competitor analysis and market trends
- Head of Content: drafts and schedules posts and articles
- Head of Finance: reconciles subscriptions and tracks runway
- One morning message triggers all four agents running in parallel
- Beyond role-based: split open specialists (messy unbounded work) from closed specialists (repeatable pipelines)

## Tradeoffs
- Cross-profile communication requires explicit handoff mechanisms
- Each profile accumulates its own token costs
- Profile proliferation can create coordination overhead that outweighs specialization benefits
