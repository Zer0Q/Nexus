# Knowledge Graph Evolution

## Definition
Building a queryable knowledge graph from notes and documents, where entities and relationships are extracted, stored, and evolved over time through automated and manual curation.

## Why It Matters
Flat note systems lose connections. Knowledge graphs surface hidden relationships between concepts, enabling discovery and synthesis that linear browsing cannot achieve.

## Key Ideas
- Entities extracted from notes become graph nodes
- Relationships become edges with types
- Query the graph for connections between concepts
- Graph evolves as new notes are added
- Can be stored in SQLite, Neo4j, orObsidian backlinks

## Tradeoffs
- Entity extraction requires NLP or manual effort
- Graph can become noisy without curation
- Query language has learning curve
