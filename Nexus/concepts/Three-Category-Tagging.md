# Three-Category Tagging

## Definition
A tagging system with three prefixed categories: topic tags (no prefix, what the note is about), status tags (status/ prefix, workflow position), and project tags (project/ prefix, project association). Each category serves a distinct retrieval purpose.

## Why It Matters
Solves the two extremes: no tags (unsearchable) vs. too many tags (noise). The prefix system means searching a tag tells you which dimension you're filtering by. Combined with search, covers almost every retrieval scenario.

## Key Ideas
- Topic tags: #productivity, #machine-learning -- what it's about
- Status tags: #status/active, #status/waiting -- workflow position
- Project tags: #project/website-launch -- project association
- Rule: only create tags used on 5+ notes, otherwise it's noise
- Combined with type filter and date range: covers almost every retrieval scenario
- Three categories, infinite filtering combinations

## Tradeoffs
- Requires consistent tag application
- Tag taxonomy needs periodic review (quarterly vault review)
- Risk of tag proliferation without the 5-note rule
