# High-Performance HMI Overview

## Summary
High-Performance HMI replaces poorly designed, colorful, cluttered HMIs with standardized, information-driven interfaces based on ISA-101. Key principles: grayscale backgrounds with color as attention-getter (48% improvement in detecting abnormal situations), information over raw data, simplified graphics, and four-level display hierarchy. Poorly designed HMIs have contributed to industrial accidents.

## Core Concepts
- [[concepts/High-Performance-HMI]] -- standardized HMI design: grayscale base, color for abnormal, information over data, simplified graphics
- [[concepts/Grayscale-HMI-Design]] -- grayscale backgrounds with color reserved exclusively for abnormal conditions
- [[concepts/Display-Hierarchy]] -- four-level structure: situational awareness, unit overview, equipment detail, diagnostics
- [[concepts/Operator-Situational-Awareness]] -- operator's ability to perceive, comprehend, and project process state
- [[concepts/Information-over-Data]] -- presenting contextual meaning (normal range indicators) rather than raw values

## Key Insights
- Poorly designed HMIs: broad colors, unnecessary graphics, visual distractions, inconsistent navigation
- Consequences: "running by the alarms" (reacting without understanding root cause), avoidable upsets, industrial accidents
- Grayscale + color for abnormal: 48% improvement in detecting abnormal situations before alarms
- "No color = normal plant" — quick glance confirms normal operations
- ISA-101 published in 2015 after 12 years of development (tasked in 2003)
- Embedded trends on important variables allow proactive adjustments before spec deviations
- Status colors: dark gray = stopped, white = running, medium gray = no feedback
- Operators are "eager and excited" to implement High-Performance HMI when shown the concepts
