# Knowledge Management Index

## Overview
Concepts, frameworks, and tools for building and maintaining personal and organizational knowledge systems.

## Core Concepts
- [[LLM-Wiki-Pattern]] -- LLM builds/maintains persistent wiki from raw sources
- [[Write-Time-Knowledge-Systems]] -- compile knowledge at ingest, not query time
- [[Query-Time-Knowledge-Systems]] -- retrieve from raw docs at query time (RAG)
- [[Tacit-Knowledge]] -- expert judgment embedded in experience
- [[Knowledge-Graph-as-Semantic-Layer]] -- structured relationships for AI reasoning
- [[Codify-without-Flattening]] -- preserve decision context, not just rules
- [[Knowledge-Mapping]] -- identifying where expertise resides
- [[Compounding-Knowledge-Context]] -- system value increases non-linearly over time
- [[Feedback-Loop-Knowledge-System]] -- vault pushes insights, doesn't just store
- [[Anti-Graveyard-Capture]] -- saving answers back into the KB

- [[Four-Layer-Vault-Architecture]] -- capture, pipeline, storage, intelligence
- [[Three-Layer-Architecture]] -- knowledge, intelligence, automation
- [[Business-Operating-System]] -- vault as continuous business processor

- [[Retrieval-First-Organization]] -- organize for finding, not storing
- [[PARA-Method]] -- Projects, Areas, Resources, Archive
- [[Date-Prefixed-Naming]] -- YYYY-MM-DD-TYPE-TOPIC convention
- [[Three-Category-Tagging]] -- topic, status, project tags

- [[Quarterly-Vault-Review]] -- periodic vault maintenance
- [[Inbox-First-Capture]] -- everything lands in inbox first
- [[Shared-Directory-Pattern]] -- agent and vault in same folder
- [[Graph-Based-Knowledge-Discovery]] -- AI follows backlinks across vault
