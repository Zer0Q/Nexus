# Enterprise AI Architecture Index

## Overview
Concepts and patterns for building AI systems at enterprise scale -- context layers, agent logic, governance, and compounding knowledge.

## Core Concepts
- [[concepts/Enterprise-Context-Layer]] -- shared substrate of knowledge, expertise, and norms for AI agents
- [[concepts/Context-Substrate]] -- three parts: data + KG, semantics/ontology, skills
- [[concepts/Skills-as-Primitives]] -- skills are to procedural knowledge what functions were to software logic
- [[concepts/Context-Mining]] -- reverse-engineering business operations from systems and runtime signals
- [[concepts/Context-Development-Lifecycle]] -- SDLC for context assets: create, test, approve, deploy, retire
- [[concepts/Compounding-Learning-Loops]] -- traces become durable context; tenth agent inherits from first
- [[concepts/Context-Governance]] -- quality, drift, lineage, versioning, approval across all capabilities
- [[concepts/Activation-and-Retrieval]] -- translating canonical context into many local dialects
- [[concepts/Agent-Logic]] -- software primitives (KGs, algorithms, program analysis) that steer LLMs
- [[concepts/Program-Analysis-for-Agents]] -- static/dynamic analysis to focus LLM context for code tasks
- [[concepts/Knowledge-Graph-for-IT-Ops]] -- KG of IT entities with embedded tribal knowledge
- [[concepts/Policy-as-Code]] -- runtime-enforced governance independent of model prompts
- [[concepts/Context-Space-Reduction]] -- using structured analysis to reduce LLM context needs
