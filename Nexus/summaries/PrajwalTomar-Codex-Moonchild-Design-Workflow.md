# Codex + Moonchild Design Workflow

## Summary
A design-to-code workflow using Moonchild (chat-powered design canvas) connected to Codex via MCP. Build a design system first, paste a PRD, brainstorm in chat mode, generate interactive screens, then Codex reads structured design data and ships production code. Replaces the Figma-to-designer-to-developer pipeline.

## Core Concepts
- [[tools/Moonchild]] -- chat-powered design canvas with MCP connector for structured design output
- [[concepts/Design-System-First]] -- build the design system before any screen; it determines output quality
- [[concepts/PRD-to-UI]] -- feed a product requirements document directly to generate full screen designs
- [[concepts/Visual-to-Code-Generation]] -- structured design data (not screenshots) fed to code generators via MCP
- [[concepts/Interactive-Preview]] -- test product flows before any code is written
- [[concepts/Variant-Generation]] -- multiple aesthetic directions from the same design system for comparison

## Key Insights
- The bottleneck in AI-built products is the design layer, not the code layer
- Design system quality determines output quality; garbage in, garbage out
- Interactive preview catches flow issues before they reach code
- MCP connection gives Codex actual design data (components, tokens, layout) not screenshots
- Brainstorming in chat mode before generating screens produces better results than blind generation
- The Figma-to-designer-to-developer cycle is being replaced by a single unified workflow
