# Skill Library as First AI Strategy

## Summary
Hiten Shah argumenta que la primera estrategia de IA de cualquier empresa debería ser construir una biblioteca de skills derivados del trabajo repetitivo de los mejores performers. Los connectors y el acceso a datos son necesarios pero insuficientes — lo que diferencia a una empresa es el juicio encapsulado en skills privados que capturan procesos específicos, heurísticas informales y estándares de calidad acumulados con el tiempo. La ventaja competitiva no viene del modelo elegido, sino del trabajo que se enseña al modelo a hacer bien.

## Core Concepts
- [[concepts/Skill-Library]] -- colección de skills como activo estratégico de empresa; manual operativo que los agents pueden ejecutar
- [[concepts/Skills-as-Primitives]] -- los skills como primitivos de conocimiento procedimental; aquí se enfatiza que encapsulan juicio, no solo pasos
- [[concepts/Tacit-Knowledge]] -- conocimiento tácito de los mejores performers como materia prima para skills; lo invisible se hace visible y reutilizable
- [[concepts/Skill-Based-AI-Agents]] -- agents que operan mediante skills; el article argumenta que los agents sin skills solo adivinan
- [[concepts/Company-Brain-Architecture]] -- la skill library como capa de juicio sobre la capa de datos y connectors

## Key Insights
- Un agent puede leer todas las notas de ventas y aún así perder la forma de un deal — el acceso a datos no crea buen trabajo, los skills sí
- Los skills son más que prompts: un prompt dice qué hacer en un momento, un skill captura una forma reutilizable de trabajar
- La evolución: primero vinieron los connectors y APIs (acceso), luego los skills (juicio), la próxima generación combina ambos en plugins inteligentes
- Los mejores skills serán privados: el proceso de escalación, los criterios de calificación y los estándares de review son específicos de cada empresa y no se descargan de marketplaces genéricos
- El patrón de reutilización es anterior a la IA: Unix commands → shell scripts → libraries → APIs → workflows → skills (cada capa hizo reutilizable una forma diferente de conocimiento)
- Lo que cambió no es el deseo de empaquetar expertise, sino el executor: antes los humanos debían leer el playbook, ahora los agents lo cargan, usan herramientas y ejecutan
- Para identificar skills: busca los workflows donde personas experimentadas superan consistentemente al resto, pregunta qué hace diferente a la mejor persona y qué todos pasan por alto
- Una skill buena previene que el mismo error se corrija dos veces; una mejor eleva el piso para todos; una gran skill captura juicio que antes tomaba años construir
