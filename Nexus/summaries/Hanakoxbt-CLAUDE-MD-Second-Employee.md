# CLAUDE.md as Second Employee

## Summary
Thread by @hanakoxbt contrasting ephemeral chatbot usage with persistent system-level AI configuration via CLAUDE.md. The real differentiator between power users and casual users is not access to better models but persistent instruction infrastructure that compounds over time. Five-section template (Role, Style, Constraints, Workflow, Quality) for project-level AI configuration.

- [[concepts/CLAUDE-MD-as-Context-Layer]] -- persistent project-level instructions read before every session

- [[concepts/Compound-Effect-of-Persistent-Instructions]] -- rules accumulate value over time as they prevent repeated mistakes

## Key Insights
- Boris Cherny (creator of Claude Code) runs thousands of agents overnight from his phone — his agents have better instructions, not better models
- Anthropic's own engineering leads told Dario Amodei they don't write code anymore
- Claude Cowork was built in a week and a half by Claude itself
- The "skill gap" in AI is not prompting but system setup
