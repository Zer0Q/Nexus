# Theory of Constraints

## Summary
TOC is a management philosophy by Eliyahu Goldratt: any manageable system is limited by a few — usually one — constraints. Optimizing around the bottleneck (not everywhere) is the only way to improve system throughput. Built on a 5-step iterative cycle and Throughput Accounting.

## Core Concepts
- [[tools/Theory-of-Constraints]] -- system limited by weakest link; focus improvement on the bottleneck
- [[concepts/Throughput-Accounting]] -- manage via Throughput (maximize), Inventory (minimize), Operating Expense (minimize)
- [[tools/Drum-Buffer-Rope]] -- flow control: set pace at constraint, buffer before it, synchronize release
- [[concepts/Exploit-Then-Elevate]] -- squeeze existing capacity before investing in more
- [[concepts/Constraint-Types]] -- physical, market, and policy constraints

## Key Insights
- A system where all resources work at 100% capacity is inherently unstable and full of hidden inventory
- Policy constraints (rules, metrics) are often the hardest to eliminate
- Any investment outside the constraint is an illusion of improvement
- Once a constraint is broken, it moves elsewhere — the cycle repeats
