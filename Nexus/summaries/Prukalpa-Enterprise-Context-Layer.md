# Enterprise Context Layer

## Summary
An enterprise context layer is the system that turns knowledge, expertise, and norms into machine-usable context for AI. It has a core substrate (AI-ready data + knowledge graph, semantics/ontology, skills) and five capabilities (mining, development lifecycle, compounding learning loops, activation/retrieval, governance). The context layer is the shared enterprise brain every agent reaches into.

## Core Concepts
- [[concepts/Enterprise-Context-Layer]] -- shared substrate of knowledge, expertise, and norms for AI agents
- [[concepts/Context-Substrate]] -- three integrated parts: data + KG, semantics/ontology, skills
- [[concepts/Skills-as-Primitives]] -- skills are to procedural knowledge what functions were to software logic
- [[concepts/Context-Mining]] -- reverse-engineering business operations from systems of record and runtime signals
- [[concepts/Context-Development-Lifecycle]] -- SDLC for context assets: create, test, approve, deploy, retire
- [[concepts/Compounding-Learning-Loops]] -- traces become durable context; the tenth agent inherits from the first
- [[concepts/Context-Governance]] -- quality, drift, lineage, versioning, approval across all capabilities
- [[concepts/Activation-and-Retrieval]] -- translating canonical context into many local dialects (MCP, APIs, vectors, graphs)

## Key Insights
- Knowledge = business map; Expertise = how work gets done; Norms = rules of acceptable action
- Skills are the new primitive: reusable, versionable, testable units of procedural knowledge
- Memory split: working/episodic memory near agent harness; semantic/procedural memory in context layer
- Context islands (separate layers per team) break the compounding loop; consolidation is inevitable
- The companies that win will be those whose context compounds, not those with the best models
