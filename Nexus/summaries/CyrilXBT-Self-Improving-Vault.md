# Cyril XBT Self-Improving Knowledge Vault

## Summary
Guide to building a feedback-loop knowledge vault: four layers (capture, pipeline, Obsidian, Claude). Solves three failure modes: capture friction (automated via Readwise, Telegram bot, Whisper), no connection layer (Claude reads across everything), no reason to return (daily brief pushes insights). Five-folder structure (Inbox, Notes, Ideas, Projects, CLAUDE.md). Daily brief and weekly synthesis as core rituals.

- [[concepts/Four-Layer-Vault-Architecture]] -- capture, pipeline, storage, intelligence
- [[concepts/Feedback-Loop-Knowledge-System]] -- information that goes in must come back out
- [[concepts/Anti-Graveyard-Capture]] -- vault that talks back, not just stores
- [[concepts/Automated-Capture-Pipeline]] -- zero-friction information ingestion

## Key Insights
- Three failure modes: capture friction, no connection layer, no reason to return
- "A second brain that never talks back is a very organized way to forget things"
- Five folders: Inbox, Notes, Ideas, Projects, CLAUDE.md
- Daily brief: 3 connections, 1 pattern, 1 question -- runs at 6am via N8N
- Weekly synthesis: emerging thesis, contradictions, knowledge gaps, one action
- Start with five notes -- the system compounds from there
