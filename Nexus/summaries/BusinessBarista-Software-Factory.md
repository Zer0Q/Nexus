# Software Factory Framework

## Summary
Framework for understanding and building AI-powered software engineering organizations. Introduces the Software Factory Ladder (Levels 0-4) from artisan craft to autonomous production. Key criterion for factory-readiness: verifiability -- tasks with automatic success signals can be industrialized first.

## Core Concepts

- [[concepts/Software-Factory-Ladder]] -- Five-level maturity model for AI-powered engineering orgs
- [[concepts/Verifiable-Automation]] -- AI automates what you can verify, not what you can specify
- [[concepts/Agentic-Engineering-Maturity]] -- Progression from human-coded to agent-directed development
- [[tools/Industrialized-Software-Development]] -- Production-line approach to building software
- [[concepts/Software-Factory]] -- Industrialized approach to software development
## Key Insights
- Level 0 (Artisan): Everything is human. Level 1 (Assisted): AI copilot, human gates all steps. Level 2 (Delegated): Agent writes code, human reviews PR. Level 3 (Supervised): Agents monitor, fix, review, auto-merge low-risk. Level 4 (Autonomous): Full agent loop, humans decide product direction.
- Most companies are at Level 1 despite thinking they're higher
- Key metric: % of PRs shipping without human review (zero = Level 1)
- Level 2->3 is hardest jump: requires automated tests you'd bet on + internal platform
- Factory-readiness criteria: verifiable output, digital I/O, volume/repeatability, reversibility
- Engineering goes first because tests pass/fail = ground truth
- Next factories: finance (reconciliation), customer support (CSAT), then strategy/design (hardest)
