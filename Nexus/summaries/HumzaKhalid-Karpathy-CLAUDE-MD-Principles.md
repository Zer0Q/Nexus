# Karpathy CLAUDE.md Principles

## Summary
Forrest Chang distilled Karpathy's thread on Claude Code failure patterns into a CLAUDE.md file that hit 110K GitHub stars in under 3 months. Four principles: think before coding (state assumptions), simplicity first (minimum code), surgical changes (touch only what's necessary), and goal-driven execution (define success criteria upfront).

## Core Concepts
- [[concepts/CLAUDE-MD-as-Context-Layer]] -- CLAUDE.md as project-level context for coding agents
- [[concepts/Vibe-Coding]] -- AI-assisted coding where workflow matters more than model
- [[concepts/Agent-Architecture-over-Intelligence]] -- agent failures are architecture problems, not intelligence problems
- [[concepts/Prompt-Validation-Over-Construction]] -- defining success criteria and constraints matters more than prompt length

## Key Insights
- Silent assumptions are the #1 failure mode of coding agents
- Adding features, flexibility, or abstractions nobody asked for creates bloat
- Refactoring adjacent code creates collateral damage
- Weak goals like "make it work" cause the model to guess endlessly
- The 4 principles are not novel; they went viral because every developer recognized their own frustrations

## Open Questions
- How do these principles translate to non-coding agent tasks?
- Can these principles be encoded as automated validation rules in agent harnesses?
