# DamiDefi Hermes Obsidian Vault Integration

## Summary
DamiDefi connected Hermes Agent to his Obsidian vault, transforming from a passive Q&A tool into an active research partner. The key differentiator from Claude Projects is live vault memory: Hermes reads and writes markdown notes that Obsidian understands, so captures land in the vault immediately and Hermes reads them next session without manual intervention. Six weeks of connected operation produced 23 skill files, cross-month narrative detection, and real-time vault queries from Telegram.

## Key Insights
- The gap between manual context systems (Claude Projects) and live-reading agents is categorical, not incremental
- SOUL.md is not a personality file -- it is the instruction layer governing how Hermes interprets your vault
- Morning briefs moved from Claude to Hermes because Hermes connects 7-day captures to the full vault, not just the inbox
- The compounding effect happens faster with Hermes because it writes to the vault as well as reads from it
- Scoped access testing for 3 days before full vault access is a non-negotiable safety step
