# 20 GitHub Scraping Repos

## Summary
Comprehensive list of 20 web scraping repos categorized into AI-native tools (Firecrawl, Crawl4AI, Browser Use, Stagehand, Skyvern, ScrapeGraph AI, AgentQL), anti-detection tools (Hyperbrowser, Crawlee, Scrapling, Steel, Playwright), production-scale frameworks (Scrapy, Puppeteer, Selenium, Colly), and specialist tools (Katana, Browserless, Maxun, Heritrix). The real challenge in 2026 is surviving anti-bot systems, not writing scraping code.

## Core Concepts
- [[tools/Firecrawl]] -- URL-to-markdown/JSON with JS rendering, CAPTCHA handling, native Claude MCP
- [[tools/Crawl4AI]] -- LLM-pipeline-focused scraper, outputs clean structured data automatically
- [[tools/Browser-Use]] -- full browser control for AI agents; navigate, click, fill forms, extract data
- [[tools/AgentQL]] -- semantic query language for AI agents; describe what you want, not CSS selectors
- [[tools/Hyperbrowser]] -- stealth headless browser bypassing Cloudflare, DataDome, bot detection
- [[tools/Scrapling]] -- adaptive selectors and anti-bot bypass with proxy rotation
- [[concepts/Anti-Bot-Evasion]] -- surviving Cloudflare, DataDome, Kasada, PerimeterX in scraping workflows

## Key Insights
- The anti-bot market hit $11B in 2025; well-built scrapers are blocked within seconds on protected domains
- Vision-based tools (Skyvern) outperform selector-based on form-heavy sites
- Playwright is 12-17pp more reliable than vision-driven tools for DOM-accessible tasks
- AI-native scrapers output data models can reason over directly; no cleaning step needed
- Scrapy remains the most reliable at genuine scale despite being older
