# SKILL.md Anatomy and Creation Guide

## Summary
@santtiagom_ comparte su experiencia creando más de 50 Skills en Claude Code, de las cuales la mayoría eliminó a los 2 días pero las que quedaron cambiaron completamente su forma de trabajar. Explica la anatomía de una SKILL.md y cómo crear la propia.

## Core Concepts
- [[concepts/Skill-Files]] -- Documentos markdown que enseñan al modelo cómo realizar tareas específicas de forma consistente
- [[concepts/SKILL-MD]] -- Archivo obligatorio de una Skill: contiene descripción y contexto de la tarea
-  -- Problema de Claude Code: mismo modelo, mismo pedido, resultado diferente cada vez
-  -- Claude sabe hacer code review pero no sabe QUÉ revisar primero ni en qué orden
-  -- Cada conversación nueva requiere re-explicar stack, preferencias y restricciones

## Key Insights
- Creó 50+ Skills, la mayoría eliminó a los 2 días — solo las que quedaron marcaron diferencia
- Una Skill NO le enseña al modelo a hacer algo, le enseña CÓMO hacerlo a tu manera
- Analogía del chef: sabe cocinar, pero sin tus recetas cocina a su manera — la Skill es la receta
- 3 problemas que resuelve: consistencia (mismo resultado cada vez), calidad (revisar en el orden correcto), eficiencia (no repetir contexto)
- SKILL.md tiene 2 partes: descripción (qué hace) y contexto (instrucciones detalladas)
- La ventaja real: dejás de repetite — las instrucciones se escriben una vez y se aplican siempre
