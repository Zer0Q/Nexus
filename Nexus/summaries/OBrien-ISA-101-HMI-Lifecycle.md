# ISA-101 HMI Lifecycle Standard

## Summary
ISA-101 provides lifecycle-based guidelines for HMI design, implementation, and management across process, hybrid, and discrete industries. Addresses operator situational awareness, display hierarchy (4 levels), and cost reduction through standardized design. ANSI-approved in 2015 as ANSI/ISA-101.01-2015. Active working groups covering philosophy/style guides, usability, mobile HMIs, and machine HMIs.

## Core Concepts
- [[concepts/ISA-101-Standard]] -- lifecycle standard for HMI design and management
- [[concepts/High-Performance-HMI]] -- standardized HMI design with grayscale base, color as attention-getter, information over data
- [[concepts/Operator-Situational-Awareness]] -- operator's ability to perceive, comprehend, and project process state
- [[concepts/Display-Hierarchy]] -- four-level display structure from overview to diagnostics
- [[concepts/Grayscale-HMI-Design]] -- grayscale backgrounds with color reserved for abnormal conditions

## Key Insights
- Legacy PID-style graphics with broad color palettes make it hard to distinguish critical from normal
- Up to $10,000 per HMI graphics page in engineering costs — ISA-101 reduces this through standardization
- 4 display levels: L1 (overall realm of control), L2 (unit operation overview), L3 (task detail), L4 (diagnostics)
- Most plants lack L2/L3 displays — end users don't know how to design them
- Future worker generation (digital natives) won't embrace poorly designed HMIs
- WG1: Philosophy & Style Guide, WG2: Usability/Performance, WG3: Mobile HMIs, WG4: Machine HMIs
