# Obsidian System for Note Utility

## Summary
Most note-taking systems optimize for capture while ignoring use. This article presents a three-zone vault architecture with five Claude-powered workflows designed to close the gap between capturing notes and actually using them. The core metric is not note count but contribution rate -- how often notes feed decisions, writing, conversations, or actions.

## Key Insights
- The gap between capturing and using is the gap between passive collection and active synthesis
- Notes that cannot contribute to any of the four uses should not be in the vault
- The Output folder creates a visible connection between captured notes and the work they produce
- After 90 days the system learns what kind of knowledge is useful for you specifically
- The contribution rate compounds from the first processed note
