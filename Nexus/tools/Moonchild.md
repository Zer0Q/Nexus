# Moonchild

## Definition
Chat-powered design canvas that starts from a design system (not a blank prompt). Import from Figma, GitHub, or live URL; describe your brand; or drop in design inspiration. Generates full screens against your system with interactive preview. Ships MCP connector for structured design output to code generators.

## Why It Matters
Most AI design tools look decent on the first screen then fall apart on consistency. Moonchild solves consistency by anchoring to a design system. The MCP connector closes the loop by feeding structured design data directly to code generators like Codex.

## Key Ideas
- Starts from design system, not blank prompt
- PRD-to-UI: paste product requirements, get full screen designs
- Chat mode: brainstorm product before generating screens
- Interactive preview: test flows before any code is written
- Variant generation: multiple aesthetic directions from same system
- MCP connector: structured design data to Codex (not screenshots)
- Refinement loop: select screen, request changes, get v2/v3 in place

## Tradeoffs
- Design system dependency -- no system, no quality output
- Variant cost -- multiple concepts cost more but save full rebuilds
- Manual nudging -- complex animations and custom patterns need hand work
