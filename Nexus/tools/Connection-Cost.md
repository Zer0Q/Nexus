# Connection Cost

## Definition
The cumulative time and mental effort spent routing information between disconnected tools in a multi-application workflow. Every cross-tool lookup, copy-paste, or context switch adds to this cost.

## Why It Matters
Connection cost is invisible during setup but compounds over time. A five-tool stack might feel comprehensive until you need to synthesise across tools -- then you pay the tax in minutes of switching, searching, and manual linking.

## Key Ideas
- Not a function of individual tool quality but of integration gaps
- Scales with the number of tools squared (each pair needs a bridge)
- Includes: context switching, manual linking, duplicate searching, lost information
- Often the dominant cost in mature multi-tool workflows
- Eliminated by moving to a [[tools/Unified-Knowledge-Environment]]

## Tradeoffs
- Reducing connection cost may mean accepting fewer features per tool
- Single-environment systems require more upfront architecture design
