# Weekly-Note-Audit

## Definition
A weekly audit that identifies notes in Zone 2 that have not been accessed in the last 14 days and determines whether they can be connected to active work or should be flagged for archival.

## Why It Matters
Enforces the core principle: notes that do not connect to anything active are not currently useful. Without regular auditing, Zone 2 accumulates dormant notes that dilute the signal-to-noise ratio of the vault.

## Key Steps
1. Read all permanent notes in Zone 2 not accessed in the last 14 days
2. For each unaccessed note: can it contribute to any active project or decision? If yes, create a connection note pointing to it
3. Does it connect to anything in active production? If yes, add the link
4. If it connects to nothing active: flag with REVIEW tag
5. REVIEW-tagged notes get one more chance in the following week
6. If still disconnected after two consecutive audits: archive to Zone 3

## Tradeoffs
- Requires tracking access patterns (Obsidian's file explorer or plugin data)
- Risk of archiving notes that are useful but infrequently accessed
