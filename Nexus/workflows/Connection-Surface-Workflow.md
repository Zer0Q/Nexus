# Connection-Surface-Workflow

## Definition
A weekly workflow that scans all notes created or modified in the past seven days and finds meaningful connections to older notes across the vault. Focuses on non-obvious connections: same principle in different domains, contradictory claims, evidence for/against claims, and emergent patterns.

## Why It Matters
Connections you did not intentionally create become visible when Claude reads across the full knowledge base. This is the workflow that makes the vault compound -- it surfaces links between notes written about different topics in ways you did not consciously intend.

## Key Steps
1. Read all permanent notes created or modified in the last 7 days
2. For each new note, scan the entire Zone 2 vault for meaningful connections
3. Meaningful connections include: same principle in different domains, contradictory claims, evidence relationships, emergent patterns across multiple notes
4. For each connection: name both notes, describe the specific connection, explain why connecting them makes both more useful
5. Only surface genuinely non-obvious connections; skip anything already linked

## Tradeoffs
- Requires Claude to read across the full vault; may hit context limits for very large vaults
- Value increases with vault size and diversity of topics
