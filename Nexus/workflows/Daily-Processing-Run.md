# Daily-Processing-Run

## Definition
An evening workflow that processes notes accumulated in Zone 1 (Capture) during the day. For each captured note: assess usefulness, rewrite in own words if useful, create permanent note with connections, archive original capture, and log results.

## Why It Matters
Prevents Zone 1 from becoming a graveyard. The daily rhythm ensures raw captures are either promoted to Zone 2 with connections or archived immediately, keeping the active vault lean and the capture zone empty by morning.

## Key Steps
1. Process all notes in Zone 1 from the current day
2. For each note, run [[concepts/Note-Usefulness-Assessment]] against active projects, decisions, and writing topics
3. If useful: rewrite in own words, identify connections to existing permanent notes, determine most likely future use case
4. Create permanent note with idea-based title, own-words content, links, and tags
5. Archive original capture to Zone 3
6. Log what was processed, what was archived directly, and what connections were made

## Tradeoffs
- Requires daily discipline; missed days accumulate backlog
- Claude-assisted processing reduces friction but adds dependency on AI availability
