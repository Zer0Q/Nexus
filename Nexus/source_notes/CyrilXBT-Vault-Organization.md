# Cyril XBT Vault Organization Guide

## Summary
Complete guide to organizing Obsidian vaults for retrieval speed, not storage. Four retrieval dimensions: content type, creation date, topic, and status. Seven-folder structure (INBOX, NOTES, PROJECTS, AREAS, RESOURCES, ARCHIVE, SYSTEM). Date-prefixed naming convention (YYYY-MM-DD-TYPE-TOPIC). Three-category tagging (topic, status/, project/). Maps of Content for topic clusters with 20+ notes. Quarterly vault review for maintenance.

## Core Concepts
- [[Retrieval-First-Organization]] -- organize for finding, not storing
- Four Dimensional Retrieval -- type, date, topic, status as search axes
- [[PARA-Method]] -- Projects, Areas, Resources, Archive organizational framework
- [[Date-Prefixed-Naming]] -- YYYY-MM-DD-TYPE-TOPIC for chronological sorting
- [[Three-Category-Tagging]] -- topic, status, project tags with prefixes
- [[Map-of-Content]] -- hub notes linking related note clusters
- [[Quarterly-Vault-Review]] -- periodic maintenance of vault organization

## Key Insights
- Filing cabinet = storage optimization; thinking system = retrieval optimization
- 5-8 top-level folders max -- more creates navigation problems
- Date prefix auto-sorts chronologically, prevents naming conflicts
- Tag rule: only create tags used on 5+ notes, otherwise it's noise
- MOCs for topics with 20+ notes -- link, don't move
- Progressive reorganization beats starting over for existing vaults
