# Graph-Based Knowledge Discovery

## Definition
Using Obsidian's linked-note graph for AI-driven knowledge discovery. The agent follows backlinks, reads related files, synthesizes across the entire knowledge base, and returns insights that would be missed through manual searching.

## Why It Matters
Transforms note searching from keyword matching to semantic exploration. "You're not searching your notes anymore. You're asking them questions." The graph reveals connections invisible from individual notes.

## Key Ideas
- Obsidian's killer feature: linked notes as synapses between ideas
- Agent follows backlinks, reads related files, synthesizes across KB
- Obsidian's graph view shows hubs and orphans
- Enable Command line in settings for agent to access Obsidian functions directly
- Combined with AGENTS.md context: agent knows how to navigate YOUR graph

## Tradeoffs
- Graph quality depends on linking discipline
- Risk of agent following irrelevant connections
- Large graphs may exceed context window for full traversal
